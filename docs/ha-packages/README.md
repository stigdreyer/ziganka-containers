# Ziganka HA packages

Home Assistant configuration packages that get deployed manually to the HALPI2's HA `/config` volume. They're not part of any `.deb` — these are user-configurable HA YAML, version-controlled here so changes are tracked and reviewable.

## Files

| File | Deploys to (on HALPI2) | Purpose |
|------|------------------------|---------|
| [`ziganka_alerts.yaml`](ziganka_alerts.yaml) | `/config/packages/ziganka_alerts.yaml` | SignalK alert routing — Phase 1 / MVP. Phone push + opportunistic Fusion TTS. |

## Architecture

Alert routing uses **push-based triggers** via the [signalk-ha](https://github.com/msallin/signalk-ha) custom integration. When SignalK publishes a notification to `notifications.*`, signalk-ha updates `sensor.signalk_last_notification` in HA, which triggers the automation. No polling, no auth token required.

Audio plays through `media_player.halos` — the Fusion MS-BB100 stereo exposed directly by signalk-ha. TTS is generated by Wyoming Piper.

### SignalK → HA state mapping

| signalk-ha state | Internal priority | Fusion TTS |
|------------------|-------------------|------------|
| `emergency`      | emergency         | yes (vol 20) |
| `alarm`          | alarm             | yes (vol 18) |
| `warn`           | warning           | yes (vol 14) |
| `alert`          | caution           | no (IMO) |
| `normal`         | —                 | ignored (clearance) |

### Fusion AUX source

AUX input on this Fusion is `source2` (SignalK path: `entertainment.device.fusion1/output/zone1/source`).

## Deploying `ziganka_alerts.yaml` (first time)

### 1. Verify prerequisites

On the HALPI2, confirm:

```bash
# Wyoming Piper container is up
sudo docker ps | grep piper
nc -zv localhost 10200   # should succeed

# HA container is up
sudo systemctl status ziganka-homeassistant-container.service

# SignalK is reachable
curl -s http://localhost:3000/signalk
```

### 2. Install signalk-ha (HACS)

In HA: **HACS → Integrations → Explore → search "SignalK"** → install [signalk-ha](https://github.com/msallin/signalk-ha).

Then: **Settings → Devices & Services → Add Integration → SignalK**
- Host: `localhost`
- Port: `3000`
- Enable notifications: yes (default: `notifications.*`)

After setup, confirm `sensor.signalk_last_notification` exists in Developer Tools → States.

### 3. Add the Wyoming integration in HA

HA UI → **Settings → Devices & Services → Add Integration → Wyoming Protocol**.
- Host: `localhost`
- Port: `10200`

After it loads you should see `tts.piper` in Developer Tools → States.

### 4. Enable HA packages (one-time)

SSH into the HALPI2 and run:

```bash
# Find the HA config volume path
sudo docker inspect homeassistant | grep -A3 '"Mounts"'
# Returns something like: /var/lib/container-apps/ziganka-homeassistant-container/data/config

CONFIG=/var/lib/container-apps/ziganka-homeassistant-container/data/config

# Add packages directive (only if homeassistant: block doesn't exist yet)
sudo sh -c "echo -e '\nhomeassistant:\n  packages: !include_dir_named packages' >> $CONFIG/configuration.yaml"

# Create packages directory
sudo mkdir -p $CONFIG/packages
```

If `configuration.yaml` already has a `homeassistant:` block, manually add `  packages: !include_dir_named packages` under it instead.

### 5. Drop in the package file

```bash
# From your laptop (uses 1Password SSH agent):
SSH_AUTH_SOCK="$HOME/Library/Group Containers/2BUA8C4S2C.com.1password/t/agent.sock" \
  scp -o IdentitiesOnly=yes -i ~/.ssh/halpi2_key.pub \
  docs/ha-packages/ziganka_alerts.yaml pi@halos.local:/tmp/

# On the HALPI2:
sudo mv /tmp/ziganka_alerts.yaml /var/lib/container-apps/ziganka-homeassistant-container/data/config/packages/
sudo chown root:root /var/lib/container-apps/ziganka-homeassistant-container/data/config/packages/ziganka_alerts.yaml
```

### 6. Restart Home Assistant

HA UI → **Developer Tools → YAML → Restart Home Assistant** (full restart required on first deploy).

### 7. Set helpers (every deploy)

> **Important:** `initial:` values on helpers only apply on first creation. After any redeploy or reload, set these manually:

**Settings → Devices & Services → Helpers:**

- `input_text.ziganka_ma_player` → `media_player.halos`
- `input_text.ziganka_notify_target` → `notify.mobile_app_<device>` or `notify.notify` to broadcast
- `input_number.ziganka_vol_*` → adjust if defaults (20/18/14) feel wrong

### 8. Test

HA UI → **Developer Tools → Actions** → call `script.ziganka_alert_test`. Expect:
- A push notification on your phone titled "⚠ Warning" (if notify target is set)
- Spoken "Warning: This is a test of the Ziganka alert system" through the Fusion stereo

## Updating the package file

After editing `ziganka_alerts.yaml`, deploy and reload (no full restart needed):

```bash
# From laptop:
SSH_AUTH_SOCK="$HOME/Library/Group Containers/2BUA8C4S2C.com.1password/t/agent.sock" \
  scp -o IdentitiesOnly=yes -i ~/.ssh/halpi2_key.pub \
  docs/ha-packages/ziganka_alerts.yaml pi@halos.local:/tmp/

# On HALPI2:
sudo mv /tmp/ziganka_alerts.yaml /var/lib/container-apps/ziganka-homeassistant-container/data/config/packages/
```

Then: **Developer Tools → YAML → Reload All YAML**

Re-set `input_text.ziganka_ma_player` to `media_player.halos` after each reload.

## Optional: chime files

For better UX with chimes before TTS, drop three short MP3 files (~1 s each) into `/config/www/alerts/` on the HALPI2:

- `emergency.mp3` — fast, urgent (5-pulse pattern matches IMO emergency)
- `alarm.mp3` — moderate (3-pulse)
- `warning.mp3` — gentle (2-pulse)

Free maritime-ish sources: [freesound.org](https://freesound.org) (search "alarm beep"). Without these files the script plays TTS directly — works fine, just less attention-grabbing.

## Troubleshooting

```bash
# On HALPI2 — check HA logs for ziganka errors
sudo docker logs homeassistant 2>&1 | grep -i ziganka | tail -20
```

If nothing fires when an alert is raised, check:
- `sensor.signalk_last_notification` in Developer Tools → States — did it update?
- `sensor.signalk_last_notification` attributes — is `state` something other than `normal`?
- Automation traces: Settings → Automations → "Ziganka: fire alert handler on new active notification" → Traces
- `input_text.ziganka_ma_player` — must be `media_player.halos` (not `media_player.music_assistant`)
- `binary_sensor.fusion_reachable` — must be `on` for audio to play

```bash
# Synthetic alert via SignalK REST API
curl -X POST http://localhost:3000/plugins/signalk-alert-manager/alerts \
  -H 'Content-Type: application/json' \
  -d '{"priority":"warning","message":"Test from curl","path":"test.curl"}'
```
